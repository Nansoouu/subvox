# Governance

Subvox is governed by the following principles.

## Decision making

1. **Technical decisions** are made by core maintainers with community input via issues/PRs
2. **Financial decisions** (pool allocations, fee structure) are publicly documented
3. **Governance changes** require a clear proposal and community feedback period

## Roles

| Role | Responsibility |
|------|---------------|
| **Maintainer** | Reviews PRs, manages releases, oversees security |
| **Contributor** | Submits code, translations, documentation |
| **Pool operator** | Manages community pool resources (Groq, GPU) |
| **Community member** | Uses the tool, provides feedback, reports bugs |

## Pool allocation

- **70%** of platform fees go back to community pools (Groq credits, GPU compute)
- **20%** to token rewards for contributors
- **10%** to operational costs (hosting, domains)

All figures are published quarterly in `reports/`.
